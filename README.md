## 🏗️ Главный проект: Автоматизированный мониторинг системы (`00_Automated_DevOps_Infrastructure`)

Центральный проект портфолио — отказоустойчивый комплекс для сбора метрик хоста и автоматического деплоя. 

### Архитектура комплекса:
* **Агент мониторинга (Python):** Автономный фоновый скрипт (демон), использующий библиотеки `psutil` и `shutil`. Собирает телеметрию хоста (нагрузка на RAM %, свободное дисковое пространство виртуалки Ubuntu). Оснащен асинхронной системой алертинга: при достижении критических порогов нагрузки (>90%) мгновенно отправляет уведомления в Telegram через API.
* **База данных (PostgreSQL):** Контейнеризированная реляционная СУБД для постоянного хранения логов и метрик. Данные изолированы и полностью защищены от потери при перезапуске или обновлении контейнеров с помощью Docker Volumes (персистентное хранилище).
* **Веб-витрина (FastAPI):** Backend-интерфейс (REST API), который осуществляет выборку актуальных данных из базы и отдаёт собранную телеметрию в реальном времени по эндпоинту `/metrics`.

### Автоматизация и CI/CD:
Вся инфраструктура полностью контейнеризирована и упакована в **Docker Compose** с использованием изолированных кастомных сетей (`networks`) для безопасного межконтейнерного взаимодействия. Настроен автоматический GitOps-пайплайн непрерывного деплоя (CD) через **GitLab CI**: при каждом пуше в ветку `main`, раннер автоматически валидирует конфигурацию оркестрации (`docker compose config`), подключается к удаленному серверу по защищенному SSH-ключу, выполняет обновление кодовой базы (`git pull origin main`) и бесшовно перезапускает сервисы.

## 🛠️ Структура портфолио по модулям

* **`00_Automated_DevOps_Infrastructure`** — Главный инфраструктурный DevOps-проект (Docker Compose, Python, PostgreSQL, FastAPI, GitLab CI).
* **`01_Linux_Network`** — Практические задачи по настройке сетевых интерфейсов, маршрутизации, DNS, портов и базовой безопасности Linux.
* **`02_Linux_Monitoring_Base`** — Скрипты автоматизации на Bash, работа с логами операционной системы и системными утилитами мониторинга.
* **`03_SQL_Database_Design`** — Проектирование реляционных баз данных, создание таблиц, управление пользователями и правами доступа (`chown`/`chmod`/`GRANT`).
* **`04_SQL_Data_Retrieval` & `05_SQL_Advanced_Queries`** — Работа с выборками данных, написание сложных и оптимизированных SQL-запросов.
* **`06_C_Language_Basics` & `07_Team_Project_C_Ping_Pong`** — Фундаментальный бэкграунд низкоуровневого программирования. Изучение работы с памятью, указателями и процессами ОС на языке Си (базовый этап закалки инженера).
